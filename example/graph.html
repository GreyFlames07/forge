▸
Element — The core implementation unit (aggregate, entity, value object, service, or projection). What developers actually build and own.
▸
Module — A deployable artifact (service, worker, function…) that packages one or more elements. Maps to a repo and a deployment unit.
▸
Datastore — A database, cache, queue, or object store. Arrows show which modules consume it; the label is the datastore name.
▸
Subgraph border — A domain: a bounded area of responsibility that groups related modules and elements within a system.
▸
Solid arrow — A contract dependency (producer → consumer, labelled with the contract name), a datastore access, or a structural element relationship.
▸
Dashed arrow — An interaction: a direct runtime call from one element's operation to another's, as defined in the interactions registry.
