▸
Environment (outer border) — A deployment target (production, staging, …). The outer border groups the modules deployed there.
▸
Module (inner border) — A deployable artifact (service, worker, function…) that packages one or more elements. Maps to a repo and a deployment unit.
▸
Element — The core implementation unit (aggregate, entity, value object, service, or projection). What developers actually build and own.
▸
Datastore — A database, cache, queue, or object store. The connecting arrow is labelled with the datastore name.
▸
Solid arrow — A contract reference (producer → consumer, labelled with the contract name) or a structural element relationship.
▸
Dashed arrow — An interaction: a direct runtime call from one element's operation to another's, as defined in the interactions registry.
