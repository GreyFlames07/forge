Nodes

[ ]
Element

The core implementation unit — an aggregate, entity, value object, service, or projection. This is what developers build.

( )
Module

A deployable unit (service, worker, function…) that owns one or more elements and is deployed as a single artifact.

Datastore

A database, cache, queue, or other persistence layer. Arrows show which modules read from or write to it.

Contract

A versioned API contract with defined inputs, outputs, and errors. Shown on edges between producer and consumers.

Domain (subgraph)

A bounded area of responsibility grouping related modules and elements. Domains do not cross system boundaries.

Edges

Contract dependency

The labelled module produces this contract; the arrow points to each consuming module that depends on it.

Datastore usage

A module reads from or writes to the target datastore. The label is the datastore name.

Element relationship

An explicit structural dependency between two elements (depends_on, references, extends, contains, triggers).

Interaction (dashed)

A direct runtime call from one element's operation to another's, as defined in the interactions registry.
